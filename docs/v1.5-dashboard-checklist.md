# v1.5 dashboard checklist

All code changes for v1.5 (Option C cross-platform sync subscription) are landed in both repos. The remaining work is in third-party dashboards — these can't be automated from the repo because they require human authentication. Run through this list before deploying Phase 7 (notify existing buyers + flip the entitlement switch).

## 1. Stripe Dashboard (https://dashboard.stripe.com)

### Create the Dash Sync product

1. **Products → Add product**
   - Name: `Dash Sync`
   - Description: `Cross-device sync for Dash Notes — end-to-end encrypted.`
   - Tax behavior: leave as the account default

2. **Add two recurring prices** under the Dash Sync product:
   - **Monthly**
     - Type: Recurring
     - Price: `$4.99 USD`
     - Billing period: Monthly
     - Lookup key: `dash_sync_monthly` (optional but recommended)
   - **Yearly**
     - Type: Recurring
     - Price: `$47.99 USD`
     - Billing period: Yearly
     - Lookup key: `dash_sync_yearly`

3. **Copy the new price IDs** (e.g. `price_1Q...`) and set the following Vercel env vars on the DashLandingPage project:
   - `STRIPE_PRICE_SYNC_MONTHLY=price_...`
   - `STRIPE_PRICE_SYNC_YEARLY=price_...`
   - `STRIPE_PORTAL_RETURN_URL=https://dashnote.io/payment/portal-return`

### Enable Stripe Customer Portal

1. **Settings → Billing → Customer portal** → enable.
2. Functionality:
   - Customers can: cancel subscriptions, switch plans (monthly ↔ yearly), update payment method, view invoices.
3. Default redirect URL: `https://dashnote.io/payment/portal-return`
4. Save changes.

### Extend the webhook endpoint

1. **Developers → Webhooks** → click the existing endpoint that POSTs to `https://dashnote.io/api/webhook`.
2. **Add events** (alongside existing `checkout.session.completed`):
   - `customer.subscription.created`
   - `customer.subscription.updated`
   - `customer.subscription.deleted`
   - `invoice.paid` (or `invoice.payment_succeeded` if `invoice.paid` not visible)
   - `charge.refunded`
3. Save. The webhook signing secret (`STRIPE_WEBHOOK_SECRET`) is unchanged.

## 2. App Store Connect (https://appstoreconnect.apple.com)

### Subscription group + products

1. **My Apps → Dash Notes → In-App Purchases → Subscriptions**
2. Create new **Subscription Group**: `Dash Sync`
3. Localized display name (English – default): `Dash Sync`

4. Add **subscription product** #1:
   - Product ID: `io.dashnote.app.sync.monthly`
   - Reference Name: `Dash Sync Monthly`
   - Subscription Duration: 1 Month
   - Price (US): Tier 5 ($4.99 USD)
   - Localized:
     - Display Name: `Dash Sync`
     - Description: `Cross-device sync, end-to-end encrypted.`
   - **Introductory Offer**:
     - Country: All / your launch market
     - Type: Free
     - Duration: 7 Days
     - Eligible: New Subscribers
   - Review Information: screenshot of the paywall screen, sandbox test account, brief notes ("subscribe to enable sync across devices").

5. Add **subscription product** #2:
   - Product ID: `io.dashnote.app.sync.yearly`
   - Reference Name: `Dash Sync Yearly`
   - Subscription Duration: 1 Year
   - Price (US): Tier 48 ($47.99 USD)
   - Same intro offer (7-day free trial).

6. Submit both for review when ready.

### Sandbox testing

1. **Users and Access → Sandbox → Testers** → create a sandbox tester.
2. On a real iOS device, sign out of your real Apple ID for the App Store, then sign in with the sandbox tester when prompted during the in-app purchase.

### App Privacy

1. **My Apps → Dash Notes → App Privacy** → ensure:
   - Data collected: **Purchases** (linked to user identity, used for app functionality).
   - Other categories: leave as currently declared (no analytics, no tracking).

## 3. RevenueCat (https://app.revenuecat.com)

### Add the two iOS products

1. **Project → Products** → New Product
   - Identifier: `io.dashnote.app.sync.monthly`
   - Store: App Store
   - Type: Auto-renewable subscription
2. Add second product `io.dashnote.app.sync.yearly` the same way.

### Entitlement

1. **Entitlements** → ensure `sync` exists (already created in v1.4 setup).
2. Attach both new products to the `sync` entitlement.

### Offering

1. **Offerings → default**
2. Add two packages:
   - `$rc_monthly` → maps to `io.dashnote.app.sync.monthly`
   - `$rc_annual` → maps to `io.dashnote.app.sync.yearly`
3. Mark this offering as the **current** offering.

### Apple Shared Secret

1. App Store Connect → Users and Access → Keys → In-App Purchase → create/copy the **shared secret**.
2. RevenueCat → Project Settings → Apps → iOS → paste the shared secret.

### Webhook

1. **Project Settings → Integrations → Webhooks**
2. URL: `https://dash-relay.efesop.deno.net/entitlements/grant-ios`
3. Authorization header: set to whatever value you'll set on the relay as `RC_WEBHOOK_AUTH` (any random 32+ char string).
4. Save.

## 4. Resend (https://resend.com)

1. **Domains → Add Domain**: `dashnote.io`
2. Add the DNS records Resend gives you (SPF, DKIM, DMARC) to the dashnote.io DNS provider. Wait for verification.
3. **API Keys → Create API Key**: scope `Send emails` only.
4. Note the key (`re_...`).

## 5. Deno Deploy (relay)

Open the playground project for `dash-relay`. Add the following env vars (Settings → Environment Variables):

| Name | Value | Notes |
|---|---|---|
| `RESEND_API_KEY` | `re_...` | from step 4 |
| `RESEND_FROM` | `Dash Notes <auth@dashnote.io>` | sender |
| `AUTH_TOKEN_SECRET` | 32+ char random hex | `openssl rand -hex 32` |
| `RC_WEBHOOK_AUTH` | matches RevenueCat webhook auth from step 3 | |
| `ENTITLEMENT_REQUIRED` | (leave unset for now) | flip to `true` only in Phase 7 |

`ENTITLEMENT_GRANT_SECRET` should already be set (used by Mac one-time grant + sync sub grant — same secret).

Deploy the updated relay source (`server/relay.ts`, `server/auth.ts`, `server/resend.ts`, `server/entitlements.ts`, `server/sync.ts`, `server/sync-tests.ts`) via the playground.

## 7. Phase 7 — Execute the launch sequence

In order, NEVER skip:

1. Confirm both repos are deployed:
   - Deno Deploy: `curl https://dash-relay.efesop.deno.net/health` returns 200.
   - DashLandingPage: visit `https://dashnote.io/subscribe` — pricing page loads, both buttons work and redirect to Stripe Checkout.
2. **Marketing copy is live** — visit dashnote.io and verify no remaining "lifetime, no subscriptions" wording. The plan updated `app/components/PaymentSection.tsx` and `app/page.tsx`; spot-check the homepage.
3. Run notify script (dry run first):
   ```bash
   cd /Users/ollie/DashLandingPage
   DRY_RUN=true npx tsx scripts/notify-existing-buyers.ts
   ```
   Verify the email list looks right + count matches roughly your expected buyer count.
4. Real send:
   ```bash
   cd /Users/ollie/DashLandingPage
   npx tsx scripts/notify-existing-buyers.ts
   ```
   Watch Resend dashboard for delivery + bounce rate. Open inbox of a personal email if it's in the list to confirm rendering.
5. **Wait 24 hours.** Be reachable for refund requests; issue them within the grace window without argument.
6. Upload iOS build 54 (v1.5.0):
   ```bash
   PATH="/usr/local/bin:$PATH" npm run ios:archive
   PATH="/usr/local/bin:$PATH" npm run ios:export
   PATH="/usr/local/bin:$PATH" xcrun altool --upload-app -f build/App-export/App.ipa -t ios --apiKey 8NXN3W8UL2 --apiIssuer 0b2ccadb-7c1e-42e7-83ed-ee2e61b52391
   ```
   In App Store Connect: create a new v1.5.0 app store version, attach build 54, attach the two new IAP products to the version, fill in updated "What's New" text, submit for review.
7. Tag the Mac/Electron build:
   ```bash
   cd /Users/ollie/rich-text-editor
   git tag v1.5.0
   git push origin v1.5.0
   ```
   GitHub Actions builds + publishes the notarized DMG; existing Mac auto-updaters pick it up next launch.
8. Push to dashnote.io for PWA users.
9. **Flip the entitlement switch:**
   - In Deno Deploy → set `ENTITLEMENT_REQUIRED=true`. Save. New sync requests now require an active subscription via either rail (Stripe sub or iOS IAP).
10. Watch `/health` + relay logs + Stripe dashboard + RC dashboard for 72 hours. Be responsive to refund or sync-broken support requests.

## 8. Phase 8 — Verification matrix

Run through the verification matrix in the plan file. Confirm:
